  Previous instruction is the focus, generate signals and swap them at the end of the cycle. So, in the next iteration execution of the previous instruction can begin.
  
  e.g. ADD, SUB
	CC2: 
		- Place ADD in writePip[ID_EX].ins
		- Generate signals for 0 (providing ADD is the first 'proper' instruction)
		- 0 will not cause register or memory reading and writing.
	CC3:
		- Place SUB in writePip[ID_EX].ins
		- Generate signals for ADD (this is a non immediate instruction, and register signals must be generated)
		- Signals will provide gateways to operand supplying
	CC4:
		- ADD generated alu signals, so, compuatiion of the supplied operands can begin
